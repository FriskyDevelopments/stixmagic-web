# Content Structure

This document describes how content is structured and where it lives in the web repository.

## Content Sources

### Current (sample data)

Static TypeScript data files in `apps/web/app/data/`:

- `packs.ts` — `SAMPLE_PACKS: ProductPack[]` — pack catalog entries
- `assets.ts` — `SAMPLE_ASSETS: AssetPreviewItem[]` — gallery asset entries
- `masks.ts` — `masks: MaskDefinition[]` — mask catalog (from `@stixmagic/types`)

### Future (pipeline-driven)

The web repo is designed to consume:

- **Pack manifests** — JSON files produced by the `stixmagic-bot` pipeline describing available packs
- **Asset previews** — GIF and WebM preview files generated by the pipeline and served from object storage or a CDN
- **Metadata JSON** — Per-asset metadata files including format availability, tags, and pack membership

## Content Types

### `ProductPack`

Defined in `packages/types/src/index.ts`.

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique pack ID |
| `name` | `string` | Display name |
| `category` | `PackCategory` | Category enum |
| `description` | `string` | Short description |
| `previewUrl` | `string` | Preview image/GIF URL |
| `assetCount` | `number` | Total assets in pack |
| `tags` | `AssetTag[]` | Searchable tags |
| `formats` | `AssetFormat[]` | Available export formats |
| `featured` | `boolean` | Featured flag |

### `AssetPreviewItem`

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Unique asset ID |
| `name` | `string` | Display name |
| `description` | `string` | Short description |
| `previewUrl` | `string` | Preview URL |
| `formats` | `AssetFormat[]` | Available export formats |
| `tags` | `AssetTag[]` | Searchable tags |
| `packId` | `string` | Parent pack ID |
